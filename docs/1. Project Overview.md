# 1. Project Overview

# **1. Overview**

| **Kategori** | **Detail** |
| --- | --- |
| **Nama Proyek** | **VendorCheckpoint** |
| **Sub Judul** | Sistem *Self Check-In* & Verifikasi Digital Safety & Health untuk Vendor Warehouse |
| **Deskripsi Singkat** | **VendorCheckpoint** adalah aplikasi web yang menyediakan **Self Check-In publik (tanpa login)** bagi vendor/driver untuk mengisi *checklist* digital yang telah direplikasi 100% dari dokumen manual resmi. Setelah submit, sistem membuat **nomor antrean otomatis** dan mengatur status ke *Waiting for Verification*. Petugas melakukan **Verifikasi Total** (Final Judgement Approved/Rejected) melalui halaman yang terproteksi login. Sistem mendukung histori lengkap, audit trail, dan laporan harian siap audit. |

---

# **2. Objectives**

- **Digitalisasi Data Menyeluruh:** Seluruh item *checklist* manual (General & Specific) direplikasi sesuai master tanpa perubahan isi dan urutan.
- **Peningkatan Compliance:** Vendor wajib memenuhi persyaratan Health, Safety, Quality, Productivity, dan Environment sesuai kategori vendor aktif.
- **Efisiensi Proses:** Sistem menyederhanakan alur pemeriksaan dengan memastikan data vendor sudah lengkap dan terstandardisasi sebelum proses verifikasi, sehingga petugas dapat memverifikasi lebih cepat.
- **Auditability:** Semua keputusan verifikasi terekam lengkap (ID Petugas, timestamp, alasan penolakan) dan dapat diakses untuk audit harian.

---

# **3. User Pain Points**

- *Checksheet* kertas mudah hilang dan sulit ditelusuri.
- Audit manual memakan waktu dan tidak terpusat.
- Tanpa sistem, pemeriksaan spesifik (B3, APAR, uji emisi, dll.) sering terlewat.
- Pencatatan waktu kedatangan/keluar tidak presisi.
- Antrean vendor tidak terstruktur.

---

# **4. Core Solutions & Modules**

| **Fitur / Modul** | **Deskripsi** | **Penanggung Jawab** |
| --- | --- | --- |
| **Public Self Check-In** | Halaman publik tanpa login untuk memulai proses. Vendor mengisi identitas dan checklist (General + Specific). | Vendor |
| **Checklist Rendering Engine** | Checklist tampil sesuai master item per kategori vendor. | Sistem |
| **Automatic Queue System** | Sistem membuat nomor antrean **YYYYMMDD-XXX**, mengatur status awal, dan menampilkan antrean secara real-time. | Sistem |
| **Verification Module** | Petugas memverifikasi data tanpa dapat mengubah input vendor, lalu memberi keputusan Final (Approved/Rejected). | Petugas |
| **Time Log Tracking** | Check-in time otomatis saat Approved. Check-out time dicatat sekali oleh Petugas. | Sistem & Petugas |
| **Monitoring Dashboard** | Ringkasan harian: total antrean, Approved, Rejected, Waiting, belum check-out, dan Lead Time. | Petugas, Leader |
| **Reporting & Export** | Rekap harian/range tanggal dalam format Excel. | Leader, Admin |
| **Vendor Master Sync** | Sinkronisasi Vendor ID dari sistem eksternal dan manajemen kategori vendor. | Admin, Sistem |

---

# **5. Target Users**

- **Vendor (Public User):** Melakukan Self Check-In dan melihat status antrean.
- **Petugas Receiving (Verifier):** Melakukan verifikasi, check-in/out, dan mengelola antrean.
- **Leader / Section:** Monitoring dan laporan.
- **Admin:** Manajemen kategori vendor, laporan, dan sinkronisasi data.

---

# **6. MVP Scope**

### **A. Self Check-In Publik**

- Input identitas vendor/driver (Nama, Vendor ID, Nama Perusahaan).
- Validasi Vendor ID berdasarkan master.
- Checklist General + Specific tampil sesuai kategori vendor aktif.
- Submit menghasilkan antrean **Waiting for Verification**.

### **B. Verifikasi Total**

- Petugas melihat antrean lengkap (read-only).
- Final Judgement: **Approved** / **Rejected** (alasan wajib).
- Penyimpanan audit trail lengkap.

### **C. Time Log**

- **Check-In Time** otomatis pada Approved.
- **Check-Out Time** diinput sekali oleh Petugas.

### **D. Queue System**

- Nomor antrean format **YYYYMMDD-XXX**.
- Tracking status: Waiting → Approved/Rejected → Completed.
- Tampilan TV Mode untuk antrean Waiting & Approved.

### **E. Dashboard & Reporting**

- Dashboard ringkasan harian.
- Export laporan harian/range tanggal (Excel).

### **F. User & Authorization**

- Login berbasis JWT eksternal untuk Petugas/Leader/Admin.
- Role-Based Access Control (RBAC).
- Vendor tidak memerlukan akun.
